1 icedb-units
2 ===============
3 
4 This is a program designed to perform basic unit interconversions.
5 For now, only basic unit conversions between common mass, length, density, temperature
6 and frequency units is supported. Eventually, a udunits-based interface is envisioned.
7 
8 How to run
9 --------------
10 
11 Execution is relatively simple. Just run the program. If no arguments are passed,
12 then follow the prompts specifying the input quantity, input units and output units.
13 
14 The code has support for performing spectral conversions (i.e. frequency to wavelength).
15 You will be prompted if you wish to do this.
16 
17 If the units are convertible, then a value is output. If the units cannot be converted,
18 then an ICEDB_LOG_ERROR message is presented.
19 
20 ###Command-line arguments:
21 
22 | Option | Required type | Description |
23 | ------ | ------------- | ----------- |
24 | --input (-i) | float | This is the input quantity (sans units) |
25 | --input-units (-u) | string | These are the input units |
26 | --output-units (-o) | string | The output units |
27 | --spec | <none> | Pass this option if a spectral conversion is desired. |
28 
29 The input quantity, input units and output units options are also positional.
30 So, the command `icedb-units 10 m um` will convert 10 meters to micrometers.
31 **Note: Negative quantities will mess up the positional parsing. In these cases, specify the input value with -i.**
