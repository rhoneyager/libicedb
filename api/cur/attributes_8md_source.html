1 Attributes
2 ==============
3 
4 The scattering database uses attributes to hold metadata about data and files.
5 The attributes in the API library are designed to match their HDF5/netCDF
6 counterparts. Both files and datasets may be tagged with attributes.
7 
8 ## What is an attribute?
9 
10 From the NetCDF manual:
11 
12 "Attributes are used to store data about the data
13 (ancillary data or metadata), similar in many ways
14 to the information stored in data dictionaries and
15 schema in conventional database systems.
16 Most attributes provide information about a specific
17 variable. These are identified by the name (or ID)
18 of that variable, together with the name of the attribute.
19 
20 An attribute has an associated
21 file / folder / group / table, a name, a data type, a length,
22 and a value. The current version treats all attributes as
23 matrices; scalar values are treated as single-element vectors.
24 "
25 
26 ## Usage
27 
28 Each attribute is stored in an HDF5 object.
29 When read or written, an attribute is encapsulated in an icedb::Attributes::Attribute<T> template class.
30 
31 ```
32 namespace icedb {
33  namespace Attributes {
34  template <class DataType> class Attribute {
35  public:
36  typedef DataType type;
37  std::vector<DataType> data;
38  std::string name;
39  std::vector<size_t> dimensionality;
40  bool isArray() const;
41 
42  Attribute(const std::string &name);
43  Attribute(const std::string &name, DataType val);
44  Attribute(const std::string &name, std::initializer_list<size_t> dims, std::initializer_list<DataType> data);
45  };
46  }
47 }
48 ```
49 
50 Attributes have a name, dimensions, and a datatype.
51 Here is how you can define an attribute:
52 ```
53  icedb::Attributes::Attribute<double> attrDouble1(
54  "TestDouble1", { 1 }, { 2.0 });
55 ```
56 This creates an attribute, called "TestDouble1", with a double type,
57 expressed as a one-dimensional array with a single value (1) in this dimension,
58 with a value of 2.0.
59 We can write this attribute to a group using:
60 ```
61  grpTest1->writeAttribute(attrDouble1);
62 ```
63 
64 In HDF5, attributes can be multi-dimensional, but in netCDF, they may
65 have only a single dimension. Here is how to define a single-dimensional
66 vector of four floating-point values:
67 ```
68  icedb::Attributes::Attribute<float> attrFloat2(
69  "TestFloat2",
70  { 4 },
71  { 1.0f, 2.0f, 2.5f, 3.0f });
72  grpTest1->writeAttribute(attrFloat2);
73 ```
74 
75 HDF5 supports multi-dimensional attributes, but NetCDF does not. Since icedb files are
76 supposed to be accessible to either NetCDF or HDF5, we likewise do not support multi-dimensional
77 attributes. However, if you want to use these attributes in your own project, then
78 a multi-dimensional attribute may be defined this way:
79 ```
80  icedb::Attributes::Attribute<float> attrFloat3(
81  "TestFloat3",
82  { 2, 3 },
83  { 1.0f, 2.0f, 3.1f, 4.2f, 5.3f, 6.0f});
84 ```
85 
86 Also note, there is a difference between a vector (a one-dimensional array)
87 and a two-dimensional array that has one of the dimension lengths set to one.
88 For example, here is how to define a 1x3 matrix. It will not work with netCDF.
89 ```
90  icedb::Attributes::Attribute<float> attrFloat4(
91  "TestFloat4",
92  { 1, 3 },
93  { 4.2f, 5.3f, 6.0f });
94 ```
95 
96 For ease of typing, we can write attributes more easily using lines like these:
97 ```
98  grpTest1->writeAttribute(icedb::Attributes::Attribute<uint64_t>("TestInt5", { 1 }, { 65536 }));
99 ```
100 or
101 ```
102  grpTest1->writeAttribute<uint64_t>("TestInt5", {1}, {65536});
103 ```
104 
105 Attributes can be floats, doubles, signed and unsigned integers (int64_t, uint64_t, int32_t, uint32_t, int16_t, uint16_t, int8_t, uint8_t),
106 and strings of characters (std::string). Any other types are invalid. I might extend the interface to support these later on, but there isn't
107 much need to do so.
108 ```
109  grpTest1->writeAttribute(icedb::Attributes::Attribute<std::string>("TestString6", "Test string 6"));
110 ```
111 
112 Currently, there is one restriction for writing strings: only a single string can be written to
113 an attribute. So, this statement will not work:
114 ```
115 grpTest1->writeAttribute(icedb::Attributes::Attribute<std::string>("TestStringSet7", { 2 }, { "Test string 7", "TS 8" }));
116 ```
117 This will eventually be fixed.
118 
119 Of course, you will also want to be able to read attributes. The syntax is similar.
120 ```
121  icedb::Attributes::Attribute<std::string> readTS6 = grpTest1->readAttribute<std::string>("TestString6");
122  std::cout << "We just read TestString6 = " << readTS6.data[0] << std::endl;
123 ```

125 ## Functions
126 
127 The attributes have a uniform programming interface.
128 This interface allows you to read attributes, add
129 attributes, check for the existence of attributes, et
130 cetera.
131 
132 All objects that can contain attributes inherit from the CanHaveAttributes class.
133 This class provides functions for listing attributes, getting their stored types,
134 reading, writing and deleting attributes.
