3 
4 Most of the data in the scattering database is stored using tables.
5 In HDF5, these are called
6 [datasets](https://support.hdfgroup.org/HDF5/Tutor/crtdat.html#def), and
7 some dataset subsets are called [tables](https://support.hdfgroup.org/HDF5/doc/HL/RM_H5TB.html).
8 In netCDF, these objects are called
9 [variables](https://www.unidata.ucar.edu/software/netcdf/docs/netcdf_data_set_components.html#variables).
10 
11 All tables may contain attributes.
12 
13 ## What is a table?
14 
15 A table is a multidimensional array of data elements, together with supporting metadata.
16 Tables may contain different types of data, such as integers, floating point numbers and character arrays.
17 In icedb, all of the elements of a table must have the same data type. This restriction is the same as in
18 netCDF, although HDF5 has a more relaxed implementation.
19 
20 Tables are usually much larger than attributes. Unlike attributes, tables are not automatically
21 loaded into memory. Tables support partial read and write operations.
22 
23 ## Usage
24 
25 See fs-test.cpp for good usage examples.
26 
27 Creating a table is easy.
28 
29 This command will make a table with two rows and three columns.
30 The data are inserted into the table in row-major form.
31 See Table.hpp to list the methods for adding larger data into tables.
32  For these other methods, the sizing of the table may be de-coupled
33 from writing data to the tables.
34 ```
35 auto tblTest = grpTest1->createTable<int64_t>("Test_table_1", { 2, 3 }, { -2, -1, 0, 1, 2, 3 });
36 ```
37 
38 Under their current implementation, table dimensions are entirely fixed and are
39 set at creation. In the future, support for extendible dimensions will be added.
40 
41 Tables support the same data types as attributes (uint64_t, int64_t, float, double).
42 Strings are on the to-do list.
43 Unlike with attributes, tables can be entirely multi-dimensional.
44 
45 Tables can have attributes.
46 ```
47 tblTest->writeAttribute<int64_t>(icedb::Attributes::Attribute<int64_t>("TestInt7", 5));
48 tblTest->writeAttribute<int64_t>(icedb::Attributes::Attribute<int64_t>("TestInt8", { 4 }, { 1, -1, 2, -2 }));
49 ```
50 
51 For compatability with netCDF, each data table must be associated with one
52 or more "dimension scales". To create and associate these scales with a table,
53 commands like these are needed:
54 ```
55 auto tblDims1 = grpTest1->createTable<int64_t>("X_axis", { 3 }, { 9, 10, 11 });
56 auto tblDims2 = grpTest1->createTable<int64_t>("Y_axis", { 2 }, { 4, 6 });
57 tblDims1->setDimensionScale("X_axis");
58 tblDims2->setDimensionScale("Y_axis");
59 tblTest->attachDimensionScale(0, tblDims2.get());
60 tblTest->attachDimensionScale(1, tblDims1.get());
61 ```
